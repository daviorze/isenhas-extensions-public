# Security Policy

## Supported Versions

The iSenhas browser extensions are actively maintained and security updates are applied to the latest available version.

Users are encouraged to always use the most recent release.

---

---

## Responsible Disclosure

We follow a responsible disclosure process:

- Vulnerabilities will be acknowledged within **72 hours**
- Fix development will begin as soon as validated
- Public disclosure may occur after a fix is released

We kindly ask researchers to avoid public disclosure until a patch is available.

---

## Scope

This repository contains the **public client-side code** for the iSenhas browser extensions.

The following are in scope:

- Browser extension source code
- Cryptographic implementation on the client
- Data handling performed locally

The following are **out of scope**:

- Private backend infrastructure
- Internal security monitoring systems
- Non-public services

---

## Security Principles

The iSenhas project is built around the following principles:

- Client-side encryption
- Zero-knowledge architecture (optional mode)
- No storage of master passwords
- Minimal permissions for browser extensions
- Defense in depth

---

## Safe Harbor

Security researchers acting in good faith, following this policy, will not face legal action for responsible vulnerability disclosure.

We appreciate the efforts of the security community in helping keep users safe.
